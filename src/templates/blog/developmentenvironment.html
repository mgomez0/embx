I want to pause and give some details on my terminal + editor setup. Persuading the reader that working from inside the terminal generates maximum developer efficiency is not in the scope of this writing, but it is what works best for me personally. This includes some key tools:

  • iterm2
  • zsh
  • oh-my-zsh
  • neovim
  • tmux
  • ripgrep
  • fzf

dotfiles can be found here

tmux session management is incredibly useful. I usually open a session per project (for example, this blog, and my next embedded project), and I'm able to quickly switch between them. Since each session maintains tab groups, it's easy to keep logical sense of shell instances. Seamless navigation between tmux panes and vim splits is an added bonus. This feature essentially converts your terminal into a full-fledged IDE.
